
# 1. 配置加载函数

## `locate_workspace_config_path`

作用：根据 `WorkspaceRoot` 找 `.coolconfig.toml`。

```rust
locate_workspace_config_path(workspace_root) -> PathBuf
```

结果：

```text
WorkspaceRoot\.coolconfig.toml
```

---

## `locate_system_config_path`

作用：找到 `.coolsystemconfig.toml`。

第一版建议从环境变量取：

```text
COOL_SYSTEM_CONFIG
```

找不到再用空配置。

```rust
locate_system_config_path() -> Option<PathBuf>
```

BAT 里以后可以这样传：

```bat
set COOL_SYSTEM_CONFIG=%~dp0.coolsystemconfig.toml
```

---

## `load_optional_cool_config`

作用：加载一个 Cool 配置文件。

```rust
load_optional_cool_config(path) -> Result<CToolScopeFileConfig, CToolScopeConfigError>
```

规则：

```text
文件不存在：返回空配置
文件存在但格式错误：返回 Error，CTool 不启用
```

---

## `parse_cool_config_toml`

作用：把 TOML 文本解析成配置。

```rust
parse_cool_config_toml(text) -> Result<CToolScopeFileConfig, CToolScopeConfigError>
```

---

## `empty_scope_file_config`

作用：生成默认空配置。

```rust
empty_scope_file_config() -> CToolScopeFileConfig
```

等价于：

```toml
[ctool_scope]
visible_paths = []
hide_paths = []
protected_paths = []
```

---

# 2. CToolScopeContext 构造函数

## `build_ctool_scope_context`

这是核心初始化函数。

```rust
build_ctool_scope_context(
    workspace_root,
    base_scope,
    system_config_path,
) -> Result<CToolScopeContext, CToolScopeConfigError>
```

它负责：

```text
1. canonicalize WorkspaceRoot
2. 加载 .coolsystemconfig.toml
3. 加载 WorkspaceRoot\.coolconfig.toml
4. 规范化配置里的路径
5. 构造 CToolScopeContext
```

如果这里失败：

```text
CTool 不注册
```

---

## `normalize_scope_file_config`

作用：把配置里的路径统一变成可判定的路径规则。

```rust
normalize_scope_file_config(
    config,
    workspace_root,
) -> Result<CToolScopeFileConfig, CToolScopeConfigError>
```

注意：

```text
相对路径：相对于 WorkspaceRoot
绝对路径：保持绝对路径
```

系统配置里的路径也可以先按 WorkspaceRoot 解析。  
以后如果需要“相对于系统配置文件目录”，再扩展。

---

# 3. 路径规范化函数

## `resolve_user_path`

作用：把用户/模型传入的路径变成绝对路径。

```rust
resolve_user_path(ctx, path) -> PathBuf
```

规则：

```text
相对路径：WorkspaceRoot + path
绝对路径：直接用
```

---

## `canonicalize_existing_path`

用于读、写、删、移动源路径。

```rust
canonicalize_existing_path(path) -> Result<PathBuf, CToolError>
```

要求路径必须存在。

---

## `canonicalize_parent_for_new_path`

用于创建文件、创建目录、移动目标路径。

```rust
canonicalize_parent_for_new_path(path) -> Result<PathBuf, CToolError>
```

要求父目录存在，但目标本身可以不存在。

---

## `path_matches_rule`

判断一个路径是否命中某条规则。

```rust
path_matches_rule(path, rule_path) -> bool
```

规则：

```text
如果 rule_path 是文件：路径相等才命中
如果 rule_path 是目录：路径等于它，或者在它下面，才命中
```

实现上可以统一用：

```text
path == rule_path || path.starts_with(rule_path)
```

---

## `matches_any_path`

判断是否命中一组路径。

```rust
matches_any_path(path, paths) -> bool
```

---

# 4. 视野判定函数

这是最重要的一组。

## `is_visible_by_base_scope`

判断是否符合 `CToolBaseScope` 默认视野。

```rust
is_visible_by_base_scope(ctx, path) -> bool
```

第一版只实现：

```text
None => false
Workspace => path 在 WorkspaceRoot 内
SelectedOnly => false，暂不支持
TheEyeofProvidence => false，暂不支持
```

---

## `can_read_path`

判断能不能读。

```rust
can_read_path(ctx, path) -> bool
```

严格按你的优先级：

```text
1. 命中 system.hide_paths => false
2. 命中 system.visible_paths => true
3. 命中 workspace.hide_paths => false
4. 命中 workspace.visible_paths => true
5. 命中 CToolBaseScope 默认范围 => true
6. 否则 false
```

---

## `can_search_path`

搜索本质是读目录/读文件。

```rust
can_search_path(ctx, path) -> bool
```

第一版可以直接等价于：

```rust
can_read_path(ctx, path)
```

以后如果你想搜索更严格，可以单独扩展。

---

## `is_hard_protected_config_path`

硬编码保护 `.coolconfig.toml` 和 `.coolsystemconfig.toml`。

```rust
is_hard_protected_config_path(ctx, path) -> bool
```

建议规则：

```text
1. canonical path 等于 workspace_config_path => true
2. canonical path 等于 system_config_path => true
3. 文件名等于 .coolconfig.toml => true
4. 文件名等于 .coolsystemconfig.toml => true
```

这样更安全。

---

## `is_protected_path`

判断是否只读。

```rust
is_protected_path(ctx, path) -> bool
```

规则：

```text
1. 命中 system.protected_paths => true
2. 命中 workspace.protected_paths => true
3. 命中硬编码配置文件保护 => true
4. 否则 false
```

---

## `can_write_path`

判断能不能写。

```rust
can_write_path(ctx, path) -> bool
```

规则：

```text
1. can_read_path(path) 必须为 true
2. is_protected_path(path) 必须为 false
3. 否则 true
```

---

## `can_create_path`

创建文件/目录稍微特殊，因为目标可以不存在。

```rust
can_create_path(ctx, path) -> bool
```

规则建议：

```text
1. 父目录必须存在
2. 父目录必须 can_write_path(parent)
3. 目标路径不能命中 hard protected
4. 目标路径不能命中 hide_paths
```

---

# 5. CTool 对外 ensure 函数

CTool 不应该直接调用 `can_xxx`，应该调用 `ensure_xxx`。  
因为 `ensure_xxx` 能返回清晰错误信息。

## 读取

```rust
ensure_read_allowed(ctx, path) -> CToolResult<PathBuf>
```

返回规范化后的 path。

---

## 搜索

```rust
ensure_search_allowed(ctx, path) -> CToolResult<PathBuf>
```

---

## 写入

```rust
ensure_write_allowed(ctx, path) -> CToolResult<PathBuf>
```

---

## 创建

```rust
ensure_create_allowed(ctx, path) -> CToolResult<PathBuf>
```

---

## 删除

```rust
ensure_delete_allowed(ctx, path) -> CToolResult<PathBuf>
```

第一版可以内部等价于：

```text
ensure_write_allowed(path)
```

但函数要单独留出来，后面方便加强。

---

## 移动

```rust
ensure_move_allowed(ctx, from, to) -> CToolResult<(PathBuf, PathBuf)>
```

规则：

```text
from 必须可写
to 必须可创建
```

目录移动工具还要额外防止：

```text
移动 WorkspaceRoot
移动到自身内部
覆盖已有目录
```

这些属于工具自身规则。

# 6. `/cs` 命令相关函数

这组是 TUI / CLI 用户命令，不是 CTool。

## `parse_ctool_scope_command`

解析 `/cs` 和 `/ctoolscope`。

```rust
parse_ctool_scope_command(input) -> Result<CToolScopeCommand, Error>
```

命令名大小写不敏感：

```text
/cs
/CS
/ctoolscope
/CToolScope
```

路径保持原样。

---

## `handle_ctool_scope_command`

执行命令。

```rust
handle_ctool_scope_command(command, ctx) -> Result<String, Error>
```

---

## `show_ctool_scope`

显示当前视野配置。

```rust
show_ctool_scope(ctx) -> String
```

输出内容建议包括：

```text
CToolBaseScope
WorkspaceRoot
.coolconfig.toml 路径
.coolsystemconfig.toml 路径
workspace visible_paths
workspace hide_paths
workspace protected_paths
system visible_paths
system hide_paths
system protected_paths
```

---

add_visible_path(workspace_config, path)

对应：

remove_visible_path(workspace_config, path)

## `add_hide_path`

add_hide_path(workspace_config, path)

remove_hide_path(workspace_config, path)

add_protected_path(workspace_config, path)

```rust

```

对应：

```text

```

保存 `.coolconfig.toml`。

```rust
save_workspace_cool_config(path, config) -> Result<(), Error>
```

注意：

```text
这个函数只能由用户命令调用。
CTool 不能调用。
```

```text
load_optional_cool_config

resolve_user_path
canonicalize_existing_path
canonicalize_parent_for_new_path

can_write_path

is_protected_path
is_hard_protected_config_path

ensure_read_allowed
ensure_search_allowed
ensure_write_allowed
ensure_create_allowed
ensure_delete_allowed

```

`/cs` 命令可以第二步做。
