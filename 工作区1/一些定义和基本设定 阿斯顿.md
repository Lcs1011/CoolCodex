# LauncherDir

LauncherDir 指的是启动器所在目录。

我们目前使用 BAT 启动 Codex，因此 LauncherDir 就是 `CodexRelease.bat` 所在的文件夹。

例如：

```text
LauncherDir = C:\Arsenal
```

LauncherDir 下会放置系统级 Cool 目录：

```text
LauncherDir\.cool-system
```

`.cool-system` 中存放系统级配置、系统级命令策略、系统级视野策略等全局安全设置。

# 文件结构
我们先临时决定，就是启动的时候使用 bat 启动 Codex
bat 存在的文件夹中的  .cool-system 文件夹里 放着系统级别的设置。
我现在 bat 先放在C:\Arsenal\ 路径里

```
C:\Arsenal\.cool-system     //系统配置文件夹
│
├─ config.toml
├─ scope.toml
├─ command.toml
│
├─ logs
└─ state
```


而我们 CMD 呼叫 Codex的文件夹 就叫做 LaunchDir
LaunchDir 文件夹里  的.cool文件夹  面放的是  项目级别的设置

```
LaunchDir\.cool    //项目配置文件夹
│
├─ config.toml       普通项目配置
├─ scope.toml        文件/文件夹视野策略
├─ command.toml      命令风险策略
│
├─ cache
├─ logs
└─ state
```


.cool-system  和 .cool
都是CTool 工具 内部写死的，不可见 不可读写
也就是说 在 Safe Mode 开启的时候 ，使用我们的Cool 开头的 permission profile 
Codex 是绝对无法看到 .cool-system  和 .cool
因为 Codex 的阅读工具 是 都是CTool 而 CTool 工具上直接写死。


```
1. 捕获 LaunchDir
2. 找 CoolSystemDir = C:\Arsenal\.cool-system
3. 找 CoolProjectDir = LaunchDir\.cool
4. 读取 .cool-system/config.toml
5. 读取 .cool/config.toml，确定 CoolWorkspace
6. 读取 .cool-system/scope.toml + .cool/scope.toml
7. 读取 .cool-system/command.toml + .cool/command.toml
8. 构造最终 CToolScope 和 CommandPolicy
```


# CoolWorkspace

LaunchDir 中 .cool 的 config.toml     里设置

如果没有设置默认等于 LaunchDir


注意 CoolWorkspace 一个重要的功能是  Command request 
都是基于 CoolWorkspace 这个路径来运行的 而不是基于 LaunchDir


这样设计的好处是   同一个工作区域这样子，我可以在不同的地方用不同的 codecs 来，用不同的方式去操作。
工作区域不再和 LaunchDir 绑定
