
## CoolBot 执行 CoolOperationDollarSyntax的显示规则

收到指令开始执行第一行显示
```text
⭐⭐⭐⭐⭐💰💰💰💰💰
```

之后每一个指令 大指令就用 
```
💰💰
<指令内容，大指令包含子指令的话这里不必显示>
⭐⭐
<执行情况，没有这里就空着就好>
⭐⭐
<执行是否成功 ，没有就空着就好>
```
对于修改文件大指令，这两个地方就应该是空的。该显示的都应该放到小指令里

小指令就用
```
💰💰💰
<指令内容，大指令包含子指令的话这里不必显示>
⭐⭐⭐
<执行情况，没有这里就空着就好>
⭐⭐⭐
<执行是否成功 ，没有就空着就好，失败要写失败原因>
```

指令如果是RunCMD 则后面加上 `[>_]`
比如
```
💰💰[>_]
```

如果路径涉及到 钱袋子后外则要加上 🚨
例如
```
💰💰🚨
```
或
```
💰💰💰🚨
```

同时出现 `[>_]` 和  🚨 ， 🚨在后
例如
```
💰💰[>_]🚨
```


最后执行完毕总结
以下面这个为开头
```
⭐💰⭐💰⭐
```

⭐⭐
是否执行完毕
一共多少个指令
执行到第几个指令
哪些指令执行中出现了报错

如果没执行完毕 报告原因（执行完毕旧略过）
不是DollarSyntax、有语法错误被拦截了同样要报告
如果是某个指令导致的 把这个指令列出来

这个设计是为了有些时候，有的指令虽然会产生执行错误，但是后续指令还会继续执行的情况

⭐⭐
执行结束后的 Action是什么
执行情况如何
